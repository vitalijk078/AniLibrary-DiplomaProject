# AniLibrary — сайт для просмотра аниме

Дипломный проект. Каталог аниме на основе Shikimori API с авторизацией, личным кабинетом и онлайн-просмотром видео с собственного сервера.

## Технологии

| Часть | Стек |
|---|---|
| Frontend | React 19, React Router 7, Axios |
| Backend | Node.js, Express 5 |
| База данных | PostgreSQL |
| Авторизация | JWT (jsonwebtoken) + bcryptjs |
| Документация API | Swagger UI (OpenAPI 3.0) |
| Данные об аниме | Shikimori API |

## Возможности

- Каталог, поиск, фильтры, страницы аниме и персонажей (данные Shikimori)
- Регистрация и вход (по email или нику), JWT-токен живёт 7 дней
- Личный кабинет: профиль, избранное, история просмотров
- Вкладка «Смотреть» — список тайтлов, у которых на сервере есть видео
- Просмотр видео — **только после авторизации** (страница и API защищены)
- Серверное кэширование ответов Shikimori (заголовок `X-Cache: HIT/MISS`)
- Swagger-документация: http://localhost:5000/api-docs

## Установка и запуск

### 1. База данных

Установите PostgreSQL и создайте базу:

```sql
CREATE DATABASE anime_db;
```

Настройки подключения лежат в `backend/.env` — поменяйте `DB_USER`, `DB_PASSWORD` и т.д. под себя. Таблицы (`users`, `favorites`, `watch_history`) создаются автоматически при первом запуске сервера.

### 2. Backend

```bash
cd backend
npm install
npm start
```

Сервер поднимется на http://localhost:5000, Swagger — на http://localhost:5000/api-docs.

### 3. Frontend

```bash
cd frontend
npm install
npm start
```

Сайт откроется на http://localhost:3000.

## Запуск в Docker (продакшен, домен haze.su)

На сервере с Linux должны быть установлены Docker и Docker Compose, а A-запись домена haze.su должна указывать на IP сервера.

```bash
docker compose up -d --build
```

Поднимутся три контейнера:

| Контейнер | Что делает |
|---|---|
| `db` | PostgreSQL 16, данные хранятся в volume `pgdata` |
| `backend` | Node.js API на порту 5000 (наружу не открыт) |
| `nginx` | Собирает React и раздаёт его на порту 80, проксирует `/api` и `/api-docs` на backend |

После запуска сайт доступен на http://haze.su, Swagger — на http://haze.su/api-docs.

Как это устроено:

- Фронтенд собирается внутри Docker с переменной `REACT_APP_API_URL=/api`, поэтому в продакшене запросы идут на тот же домен, а nginx перенаправляет их на backend. При локальной разработке переменной нет — используется `http://localhost:5000/api`.
- Папка `backend/videos` примонтирована в контейнер как volume: видеофайлы можно добавлять прямо на сервере, пересборка не нужна.
- Backend ждёт готовности базы через healthcheck, поэтому контейнеры можно запускать одной командой.
- В nginx включён `proxy_buffering off` для location `/api` — видео начинает отдаваться сразу, а перемотка (заголовок Range) проходит через прокси.

Полезные команды:

```bash
docker compose logs -f backend
docker compose restart backend
docker compose down
```

## Как добавить видео

1. Возьмите ID тайтла из адресной строки: для страницы `http://localhost:3000/anime/57555` это `57555`.
2. Положите видеофайл в папку `backend/videos` и назовите его этим ID: `57555.mp4`.
3. Готово — тайтл появится на вкладке «Смотреть», а на его странице возникнет кнопка «Смотреть онлайн».

Поддерживаемые форматы: `.mp4`, `.webm`, `.m4v` (то, что умеют играть браузеры). Перезапускать сервер не нужно — папка сканируется при каждом запросе.

## Как работает просмотр видео

1. Вкладка «Смотреть» запрашивает `GET /api/videos`: сервер читает имена файлов в папке `videos`, собирает из них список ID и одним запросом получает у Shikimori информацию по всем тайтлам сразу (ответ кэшируется).
2. На странице просмотра защищённый эндпоинт `GET /api/watch/:id` проверяет токен, отдаёт формат и размер файла и записывает просмотр в историю.
3. Сам файл отдаёт `GET /api/stream/:id` через `res.sendFile()`. Express сам поддерживает заголовок `Range`, поэтому перемотка работает: браузер запрашивает кусок файла и получает ответ `206 Partial Content`.
4. HTML-тег `<video>` не умеет отправлять заголовок `Authorization`, поэтому для него токен передаётся параметром `?token=...` — middleware `auth.js` принимает оба варианта.

## Как работает кэширование (backend/cache.js)

Кэш — это обычный JavaScript `Map`, который живёт в оперативной памяти сервера.

1. Перед запросом к Shikimori сервер собирает **ключ**: путь запроса + параметры, например `/animes{"page":1,"limit":20}`.
2. Если по этому ключу в `Map` есть запись и её время жизни не истекло — данные отдаются **из памяти**, без обращения к Shikimori. В ответе будет заголовок `X-Cache: HIT`.
3. Если записи нет (или она устарела) — сервер идёт к Shikimori, сохраняет ответ в `Map` вместе с временем истечения (`expiresAt = сейчас + TTL`) и отдаёт клиенту. Заголовок — `X-Cache: MISS`.
4. TTL (время жизни) разный: списки аниме — 5 минут, детали тайтла — 30 минут, жанры — сутки.

Зачем это нужно: повторные запросы отдаются за ~1 мс вместо ~1 секунды, а Shikimori не банит за частые обращения (у них лимит 5 запросов/сек).

Проверить вживую: открыть DevTools → Network, обновить главную страницу дважды и посмотреть заголовок `X-Cache`. Количество записей в кэше: http://localhost:5000/api/cache/stats.

## Как работает авторизация

1. При регистрации пароль хэшируется (`bcrypt`) и сохраняется в PostgreSQL — в открытом виде пароли не хранятся.
2. После входа сервер выдаёт JWT-токен, подписанный секретом из `.env`. Фронтенд хранит его в `localStorage` и подставляет в заголовок `Authorization: Bearer <токен>` через interceptor в `src/api.js`.
3. Защищённые маршруты (`/api/watch/:id`, `/api/stream/:id`, избранное, история) проходят через middleware `backend/middleware/auth.js`: он проверяет подпись токена и кладёт данные пользователя в `req.user`. Нет токена или он невалиден — ответ 401.
4. На фронтенде страницы `/watch/:id` и `/profile` обёрнуты в `ProtectedRoute` — неавторизованного пользователя перекидывает на `/login`, а после входа возвращает обратно.

## Структура backend

```
backend/
├── index.js            точка входа, подключение роутов и Swagger
├── db.js               подключение к PostgreSQL + создание таблиц
├── cache.js            кэш в памяти (Map + TTL)
├── shikimori.js        запросы к Shikimori через кэш
├── swagger.js          описание API (OpenAPI 3.0)
├── videos/             сюда вручную кладутся видеофайлы вида 57555.mp4
├── middleware/
│   └── auth.js         проверка JWT-токена (заголовок или ?token=)
└── routes/
    ├── auth.js         регистрация, вход, текущий пользователь
    ├── users.js        избранное, история просмотров
    ├── animes.js       прокси Shikimori с кэшированием
    └── videos.js       список видео, информация и потоковая отдача файлов
```
